# Listing media checklist

Fab listings convert better with clear **workflow** and **UI-visible** proof. Prepare these before you submit for review.

## Required / strongly recommended

| Asset | Spec / notes |
|-------|--------------|
| **Thumbnail** | Clear read at small size; show the plugin panel + a busy level. Avoid tiny unreadable text. |
| **Hero / cover** | Same story as thumbnail at higher resolution per Fab’s current image guidelines. |
| **Gallery image 1** | Dockable panel with several **named selection sets** listed. |
| **Gallery image 2** | Editor: actors selected in the viewport after **Recall** from a set. |
| **Gallery image 3** | Before/after: chaotic outliner vs focused selection (same viewpoint if possible). |
| **Gallery image 4** | Saving a new set from the current selection (dialog or inline name field). |
| **Trailer (45–75 s)** | Narrated or captioned: select actors → save set → clear selection → recall set → batch edit (e.g. move or tweak one property). Show real editor footage. |

## Content tips

- Use a **demo map** built for the separate example project (see `example-project/README.md`).  
- Use **captions** so buyers understand each step without audio.  
- Avoid NDA client content; use original or licensed demo assets only.

## Safe dimensions

Follow **Fab’s current image and video specifications** in the publisher portal when uploading (sizes change; verify at submission time).
