# Listing media checklist

Documentation products sell better when buyers **see structure** and **trust** the author.

## Recommended assets

| Asset | Notes |
|-------|--------|
| **Thumbnail** | Clean title + “UE5 · Docs & templates”; avoid tiny unreadable text. |
| **Hero / cover** | Table-of-contents style preview or stylized folder tree. |
| **Gallery 1** | Screenshot of `DOCUMENTATION.md` index or a guide page (readable zoom). |
| **Gallery 2** | Before/after: chaotic `Content/` vs. organized convention table from the docs. |
| **Gallery 3** | Git/LFS diagram or sample `.gitattributes` excerpt. |
| **Trailer (60–90 s)** | Voice or captions: who it’s for → what’s inside → how to apply in week one → CTA to readme. |

## Tips

- Show **real Markdown** in a code-friendly font—buyers want to feel the depth.  
- Avoid claiming official Epic endorsement.  
- If you localize, state languages in the listing.

Follow **Fab’s current image and video specs** in the publisher portal.
