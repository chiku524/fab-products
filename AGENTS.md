## Learned User Preferences

- Target Unreal Engine 5.4 and newer for plugin work and Fab-related packaging in this workspace.
- Prefer new Fab products as Unreal Engine plugins.
- When wrapping tasks, appreciates explicit next steps (including validation and git workflow) rather than open-ended handoffs.
- Prefers simple, step-by-step Unreal testing and setup guidance; is not deeply familiar with day-to-day UE editor workflows.
- Expects the agent to run concrete terminal or CLI commands (not only describe them) when they unblock local or Unreal work on Windows; Epic’s “Generate Visual Studio project files” on `.uproject` may be missing from Explorer or IDE sidebars—use `UnrealVersionSelector.exe /projectfiles` or `level-selection-sets/scripts/generate-ue-project-files.ps1` instead of assuming a working right-click menu.
- For editor plugins such as Workflow Toolkit, values polished Slate UX (grouped cards, comfortable spacing, multi-column layouts when helpful, and cohesive styling/icons within what the engine supports) and keyboard shortcuts exposed through Unreal’s standard flow (Editor Preferences → Keyboard Shortcuts, context “Workflow Toolkit”).
- After C++/Slate changes to a plugin, expects a full plugin rebuild (for example `RunUAT.bat BuildPlugin`) and often a full editor restart before panel UI updates appear; hot reload alone is not assumed sufficient.

## Learned Workspace Facts

- **Fab listing checklist** for all products: **`docs/fab-listing-index.md`** (license, `FAB_LISTING*`, `SUPPORT`, `PACKAGING*`, `TESTING` / verification).
- Fab-oriented Unreal products in this repo use per-product folders with documentation (including `docs/TESTING.md` for editor verification), license, and packaging-oriented docs (for example listing copy and support notes).
- For Fab listing and product metadata in this workspace, use the publisher name Immersive Labs.
- Unreal plugin layout for test projects is `YourProject/Plugins/<ProductFolder>/` (for example `LevelSelectionSets` from `level-selection-sets/LevelSelectionSets/`); there is no generic folder named `plugin`, which often confuses first-time setup.
- **Fab marketplace (RunUAT) plugin zips:** From a machine with the target engine installed, run **`fab-products/scripts/build-fab-marketplace-drops-ue57.ps1`** (defaults: **UE 5.7** at `C:\Program Files\Epic Games\UE_5.7`, output to **`C:\ilfab-ue57\...`** to avoid Windows **MAX_PATH** issues, then **`fab-products/fab-marketplace-drops/UE5.7-Win64/*.zip`**). The script packages **Harbor Suite**, **Level Selection Sets**, **World Builder Templates**, **Workflow Toolkit**, and **WorldBuilder Audit & Convert**. **Optional:** `verify-fab-prerequisites.ps1` for static + optional build-folder checks. For **Immersive Labs UE bundle** staging from the same build, use **`immersive-labs-ue-bundle/scripts/stage-bundle.ps1 -MonorepoBuildRoot "C:\ilfab-ue57"`** (see **`PUBLISHER_ASSEMBLY.md`**). Use each product’s **`docs/TESTING.md`**, **`VERIFICATION.md`**, or **`FAB_LISTING*.md`** before publishing.
- **Harbor-style demos** for Level Selection Sets, World Builder Templates, and Workflow Toolkit live next to their product trees: **`LevelSelectionSetsDemo/`**, **`WorldBuilderTemplatesDemo/`**, **`WorkflowToolkitDemo/`** each include **`Plugins/<Product>/`** (full source + content), **`LICENSE`**, **`Content/Maps/README.md`**, and the same **`.uproject` + `Source/`** pattern as **`Harbor/`**. After editing the canonical plugin folder, run **`scripts/sync-fab-demo-plugins.ps1`** at the **fab-products** root to refresh the embedded plugin copy. The **immersive.labs** public **Fab plugin** downloads use RunUAT zips from **`build-fab-marketplace-drops-ue57.ps1`**, copied into the site with **`immersive.labs/scripts/sync-fab-plugin-zips-to-web.ps1`** (see that repo’s **`apps/web/public/plugin-packages/UE5.7-Win64/`**). **`apps/web/src/data/fabPluginPackages.ts`** in that repo is the canonical **`zipFile`** list and in-zip packaged-root labels for **`/fab-products`**—keep them aligned with RunUAT output. Optional **source sample project** archives for local use: **`immersive.labs/scripts/package-fab-product-zips.ps1`** (outputs to **`public/fab-samples/`** by default).
- `level-selection-sets/scripts/generate-ue-project-files.ps1` regenerates Visual Studio project files for a `.uproject` via Epic’s `UnrealVersionSelector.exe`, useful when Windows shell integration for `.uproject` is broken or absent.
- For hands-on testing on Windows, separate Unreal projects for this repo’s plugins live under `Documents/Unreal Projects/`, typically one folder per plugin (for example LevelSelectionSets, WorldBuilder, `HarborSuite` for Harbor Suite, and `WorkflowToolkit` for the Workflow Toolkit plugin). That project often links the repo plugin via a directory junction to `Plugins/WorkflowToolkit`, or installs a packaged build with `workflow-toolkit/scripts/copy-packaged-plugin-to-test-project.ps1` (defaults: newest `workflow-toolkit/dist/WorkflowToolkit-UE*-Win64` from `package-plugin.ps1`, destination `Documents\Unreal Projects\WorkflowToolkit\Plugins\WorkflowToolkit`).
- **`.uproject` / `.uplugin` version fields:** A `.uproject` may list an older `EngineAssociation` than the installed editor (for example 5.4 vs 5.7); Unreal may prompt to retarget or confirm—this is normal. A plugin `.uplugin` may set `EngineVersion` to the day-to-day editor to reduce mismatch prompts; minimum Fab compatibility can still be described in product docs separately.
- In Unreal **UObject reflection headers**, include **`#include "TypeName.generated.h"`** immediately after `CoreMinimal.h` and before any `UENUM` / `USTRUCT` / `UCLASS` that uses `GENERATED_*` macros; wrong order breaks the Unreal Header Tool. Prefer includes that are on the module’s normal compile path (some `Templates/…` headers are not, depending on context).
- **Harbor Suite** (`harbor-suite/`) is the small-team editor workflow product: sample project **`Harbor/Harbor.uproject`** embeds **`Harbor/Plugins/HarborSuite/`**. **`EngineAssociation` / `EngineVersion` are pinned to 5.7** for day-to-day use. Documentation lives in **`harbor-suite/docs/`** (start with **`QUICKSTART.md`** and **`TESTING.md`**). Packaging: **`harbor-suite/scripts/package-plugin.ps1`**; VS project regeneration: **`harbor-suite/scripts/generate-ue-project-files.ps1`**. The **Harbor Annex** demo map is documented, not committed—see **`Harbor/Content/Maps/README.md`** and **`docs/DEMO_MAP_BEAT_SHEET.md`**.
