# Sample maps — World Builder Templates

Shipped levels live under `WorldBuilderTemplates/Content/Maps/`:

| Map | Role |
|-----|------|
| **WBT_Template_Outdoor** | Sun (soft source angle) / sky / clouds / bloom+grade; light height fog; **sphere reflection** capture; grid ground; path stones + terrace; trees, rocks, fills |
| **WBT_Template_IndoorBlockout** | Greybox room with **split north wall** (central gap); stairs; **rect** ceiling panels + points; **reflection** capture; pillars, bench, table |
| **WBT_Template_Ocean** | Coast **reflection** capture; island knoll + rocks; warm key + horizon rim lights; bloom + light vignette; water/shore + fog (no Water plugin) |
| **WBT_Template_City** | Plaza **reflection**; **street lamps** + median strips; towers + antennas; urban haze; corner plaza spots; bloom + vignette |

They are generated by the editor commandlet **`WorldBuilderTemplatesCreateSampleMaps`**, which calls **`NewBlankMap`**, runs **`WorldBuilderTemplatesSampleMapBuilder`** to add lights/sky/meshes, then saves each package under the plugin content path.

## Regenerating (maintainers)

1. Use a **C++** project with **WorldBuilderTemplates** enabled under `Plugins/`.
2. Close the editor.
3. Run:

   ```powershell
   cd path\to\fab-products\worldbuilder-templates\scripts
   .\create-sample-maps.ps1 `
     -EngineRoot "C:\Program Files\Epic Games\UE_5.7" `
     -UProjectPath "C:\path\to\YourProject.uproject"
   ```

4. Commit the updated `.umap` (and sidecar assets if the engine creates any) under `WorldBuilderTemplates/Content/Maps/`.

If the commandlet fails, check the log for **SaveLevel** / path errors; ensure the plugin is enabled and the project has been built at least once so the plugin mounts at `/WorldBuilderTemplates`.
