# Listing media checklist

Fab listings convert better with clear **before/after** and **UI-visible** proof. Prepare these before you submit for review.

## Required / strongly recommended

| Asset | Spec / notes |
|-------|----------------|
| **Thumbnail** | Clear read at small size; show plugin UI + environment context. Avoid cluttered text. |
| **Hero / cover** | Same story as thumbnail at higher resolution per Fab’s current image guidelines. |
| **Gallery image 1** | Audit panel with repeated actors / candidates visible in a dense scene. |
| **Gallery image 2** | **Preview** or dry-run panel showing what will change. |
| **Gallery image 3** | Before/after: manual repeats vs instanced cleanup (same viewpoint if possible). |
| **Gallery image 4** | Preset selector (Forest / City / Interior / Mobile-VR) or export dialog. |
| **Trailer (45–75 s)** | Narrated or captioned: open tool → scan → preview → convert → export. Show real editor footage. |

## Content tips

- Use a **demo map** built for the separate example project (see `example-project/README.md`).  
- Use **captions** so buyers understand each step without audio.  
- Avoid NDA client content; use original or licensed demo assets only.

## Safe dimensions

Follow **Fab’s current image and video specifications** in the publisher portal when uploading (sizes change; verify at submission time).
